## Описание проекта

QuizApp позволяет пользователю авторизоваться через Google или войти анонимно, просматривать список тем, выбирать тесты, проходить вопросы с вариантами ответов и видеть прогресс прохождения. Для наполнения базы данных используется административный веб-инструмент, связанный с Firebase, через который можно создавать и редактировать темы, тесты, пользователей и отчёты.

В рамках курсового проекта приложение рассматривается как пример архитектурно организованного клиент-серверного мобильного приложения, в котором мобильный клиент не хранит основные данные локально, а получает их из удалённого API-слоя Firebase.

## Основные возможности

- авторизация пользователя через Google;
- анонимный вход без регистрации;
- загрузка списка тем из Cloud Firestore;
- отображение тем в виде сетки карточек;
- боковое меню со списком тестов по темам;
- переход на экран выбранного теста;
- прохождение теста по шагам;
- отображение правильного или неправильного ответа;
- сохранение результата прохождения теста в Firestore;
- отображение прогресса по темам;
- экран профиля пользователя;
- просмотр количества завершённых тестов;
- обработка состояний загрузки и ошибки.

## Архитектурный подход

В проекте используется упрощённая архитектура, близкая к **MVVM + Service Layer**:

- **View** — экраны и виджеты Flutter, отвечающие за отображение интерфейса;
- **State / ViewModel-like слой** — классы состояния, например `QuizState`, управляющие прогрессом прохождения теста и выбранным ответом;
- **Service Layer** — классы `AuthService` и `FirestoreService`, изолирующие работу с Firebase;
- **Model** — классы `Topic`, `Quiz`, `Question`, `Option`, `Report`, описывающие структуру данных;
- **Backend/API** — Firebase Authentication и Cloud Firestore.

Такой подход отделяет интерфейс от доступа к данным и упрощает поддержку проекта. Экран не обращается напрямую к базе данных на низком уровне, а использует сервисы, которые инкапсулируют обращения к Firebase API.

## Работа с API и Firebase

В проекте Firebase используется как удалённый backend/API-слой. Вместо классического REST API применяется Firebase SDK, который предоставляет готовые методы для чтения, записи и подписки на изменения данных.

Основные операции API-слоя:

- `Firebase.initializeApp()` — инициализация Firebase при запуске приложения;
- `FirebaseAuth.instance.authStateChanges()` — отслеживание состояния авторизации;
- `signInWithCredential()` — вход через Google;
- `signInAnonymously()` — анонимная авторизация;
- `FirebaseFirestore.instance.collection('topics').get()` — получение списка тем;
- `FirebaseFirestore.instance.collection('quizzes').doc(quizId).get()` — получение конкретного теста;
- `snapshots()` — подписка на изменения пользовательского отчёта;
- `set(..., SetOptions(merge: true))` — обновление прогресса пользователя.

## Модели данных

В приложении используются следующие основные сущности:

### Topic

Тема тестирования. Содержит название, описание, изображение и список связанных тестов.

```dart
class Topic {
  final String id;
  final String title;
  final String description;
  final String img;
  final List<Quiz> quizzes;
}
```

### Quiz

Тест внутри выбранной темы. Содержит заголовок, описание, тему и список вопросов.

```dart
class Quiz {
  String id;
  String title;
  String description;
  String video;
  String topic;
  List<Question> questions;
}
```

### Question

Вопрос теста со списком вариантов ответа.

```dart
class Question {
  String text;
  List<Option> options;
}
```

### Option

Вариант ответа. Содержит текст ответа, пояснение и признак правильности.

```dart
class Option {
  String value;
  String detail;
  bool correct;
}
```

### Report

Отчёт пользователя о прохождении тестов. Содержит идентификатор пользователя, общее количество завершённых тестов и прогресс по темам.

```dart
class Report {
  String uid;
  int total;
  Map topics;
}
```

## Структура данных в Firestore

Для работы приложения требуется создать в Cloud Firestore следующие коллекции:

```text
topics/
  topicId/
    title: string
    description: string
    img: string
    quizzes: array

quizzes/
  quizId/
    id: string
    title: string
    description: string
    video: string
    topic: string
    questions: array

reports/
  userId/
    uid: string
    total: number
    topics: map
```

Пример логики хранения прогресса:

```text
reports/{uid}
  total: 3
  topics:
    history: ["quiz_1", "quiz_2"]
    physics: ["quiz_3"]
```

## Обработка состояний

В проекте реализованы основные состояния, необходимые для клиент-серверного мобильного приложения:

- **loading** — отображение экрана загрузки при инициализации Firebase и получении данных из Firestore;
- **error** — отображение сообщения об ошибке при неудачном получении данных;
- **empty** — сообщение о пустом списке тем, если в базе данных нет записей;
- **data** — отображение тем, тестов, вопросов и профиля после успешной загрузки данных.

## Требования для запуска

Перед запуском необходимо установить:

- Flutter SDK;
- Dart SDK, совместимый с Flutter;
- Android Studio или IntelliJ IDEA / JetBrains IDE с плагинами Flutter и Dart;
- Android SDK;
- Firebase-проект с включёнными Authentication и Cloud Firestore;

environment:
  sdk: '>=2.17.0 <3.0.0'

## Установка и запуск

Склонировать репозиторий:

```bash
git clone https://github.com/EliseMik/kp-ios.git
cd kp-ios
```

Установить зависимости:

```bash
flutter pub get
```

При необходимости пересобрать сгенерированные модели JSON:

```bash
flutter pub run build_runner build --delete-conflicting-outputs
```

Запустить приложение:

```bash
flutter run
```

## Настройка Firebase

Для запуска приложения с собственным Firebase-проектом необходимо:

1. Создать проект в Firebase Console.
2. Включить **Authentication**.
3. Включить способы входа:
    - Google Sign-In;
    - Anonymous Sign-In.
4. Создать базу данных **Cloud Firestore**.
5. Добавить Android-приложение с package name:

```text
com.qweqweqwe.quizapp
```

6. Скачать файл `google-services.json` и поместить его в каталог:

```text
android/app/google-services.json
```

7. Для iOS-сборки добавить iOS-приложение в Firebase, скачать `GoogleService-Info.plist` и подключить его в Xcode-проекте.

Если iOS-директория отсутствует или была удалена, её можно восстановить командой:

```bash
flutter create .
```

После этого необходимо повторно подключить Firebase-конфигурацию для iOS.
